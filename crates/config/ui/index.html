一般

直前に確定した読みと文字列をローカルの feedback.jsonl に追記します。 外部送信はありません（既定オフ。反映にはアプリの再フォーカスが必要な場合があります）

オンのとき、かな入力モードで打った数字を既定で全角（１２３）確定します。 変換候補で半角も選べます（既定オン。反映には新しい TIP 起動が必要な場合があります）

オンのとき、かな入力モードの「、。」を全角で確定します。オフで半角「,.」になります （既定オン。反映には新しい TIP 起動が必要な場合があります）

オンのとき、かな入力モードの記号（/ ? ~ : ; ! @ # ( ) 等）を全角（・？～：；！＠＃（）等）で入力します。 オフで半角のまま（既定オフ。「、。」は句読点の幅、長音の「ー」はこの設定と無関係。反映には新しい TIP 起動が必要な場合があります）

オンのとき、ライブ変換で漢字に置き換わって見えなくなる「今打っている読み」をキャレット上側の小窓に常時表示します。 候補ウィンドウ表示中は自動的に隠れます（既定オン。反映には新しい TIP 起動が必要な場合があります）

オンのとき、長文入力で先頭の文節が自動確定されても、読みモニタは Enter で全て確定するまで打った読みを累積表示し続けます （既定オン。切り替えの反映には新しい TIP 起動が必要な場合があります）

読みモニタの窓がこの文字数（全角換算）ぶんの幅で頭打ちになり、超えた分は先頭から隠れます （10〜100、既定34。反映には新しい TIP 起動が必要な場合があります）

かな入力中に Shift+英字を押したときの挙動です。「英語入力モード」は MS-IME 系 （Shift+A で「A」が未確定に入り、以降は小文字英字が続き、Enter で確定）。 「大文字をそのまま確定」は Google 日本語入力/ATOK 系です （反映には新しい TIP 起動が必要な場合があります）。

ターミナルや vim で「日本語モードの抜け忘れ」を防ぎます。トリガキーの変更は 「キー設定」ページの「一時かなモード開始」で行います（反映には新しい TIP 起動が必要な場合があります）。
